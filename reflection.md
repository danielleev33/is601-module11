##Reflection

This module gave me more practice working with the backend setup behind the app rather than just routes on the surface. I spent time getting the local environment, Docker services, PostgreSQL, Playwright tests, and GitHub Actions workflow working together. That ended up being a big part of the learning process.

The biggest challenges were environment/configuration related:
Docker Desktop and WSL connectivity
stale PostgreSQL volumes
container naming conflicts
dependency/version conflicts
GitHub Actions deployment secrets

Working through those issues helped me understand how the application, database, tests, and deployment pipeline connect together. It was also a reminder that a lot of software work is not just writing code; sometimes it is wrestling with the setup until it stops fighting back.
This module gave me more hands on experience with the backend side of the application, especially the parts that happen underneath the actual routes and user interface. A big part of this assignment wasn’t just writing or reading code, but making sure the full environment worked correctly across local development, Docker containers, PostgreSQL, Playwright testing, and GitHub Actions.
